You are a senior product designer and UI/UX expert.

Your task is to design a complete, modern, and production-level web application UI for a platform called:

“Insta-Study” — a peer-to-peer tutoring platform.

The system allows users to:
- register and login
- search for tutors
- view tutor profiles
- book sessions
- apply to become a tutor
- manage bookings and availability

IMPORTANT SYSTEM LOGIC:
- Every user registers as a normal user (no role selection at signup)
- A user can later apply to become a tutor
- After approval, the same user can act as both:
  - student (book sessions)
  - tutor (offer sessions)
- There should NOT be separate dashboards for student and tutor
- Use a single adaptive dashboard

-------------------------------------

🎯 DESIGN GOAL:
Create a clean, modern, minimal, and highly usable web UI similar in quality to platforms like:
- Stripe
- Linear
- Notion
- Airbnb

Focus on:
- clarity
- usability
- clean spacing
- proper hierarchy
- real-world UX patterns

-------------------------------------

📱 DESIGN TYPE:
- Web application (desktop-first)
- Responsive (should adapt to mobile, but do NOT design separate mobile screens)
- Use modern UI patterns (cards, grids, clean typography)

-------------------------------------

🎨 STYLE GUIDELINES:
- Minimal and clean design
- Soft shadows, rounded corners
- Modern gradient accents (purple/pink theme)
- Consistent spacing system
- Clear typography hierarchy

-------------------------------------

📄 DESIGN ALL THESE SCREENS (COMPLETE FLOW):

1. Landing Page
- Navbar (Home, About, Contact, Login, Signup)
- Hero section with:
  - headline
  - subtext
  - CTA buttons (Find Tutor, Become Tutor)
- Feature section (3 cards)
- Footer

-------------------------------------

2. Authentication
- Login Page
- Signup Page

-------------------------------------

3. Dashboard (Main Screen After Login)
- Welcome section
- Search bar (main focus)
- “Find Tutor” CTA
- “Become a Tutor” CTA
- Suggested tutors
- My bookings preview

IMPORTANT:
- Single dashboard (no separate student/tutor dashboards)
- Show extra options if user is a tutor

-------------------------------------

4. Tutor Search Page
- Search bar
- Filters (optional)
- Tutor cards list:
  - name
  - subject
  - rating
  - view profile button

-------------------------------------

5. Tutor Profile Page
Two states:

A) Public View (student view):
- tutor info
- subjects
- experience
- rating
- availability slots
- “Book Session” button

B) Owner View (tutor view):
- edit profile
- manage availability

-------------------------------------

6. Become a Tutor Page
- Form:
  - subjects
  - experience
  - hourly rate
- Submit application button
- Show status (Pending / Approved)

-------------------------------------

7. Availability Management Page (Tutor)
- Add time slots
- View slots
- Edit/delete slots

-------------------------------------

8. Booking Page
- Select tutor
- Select time slot
- Confirm booking

-------------------------------------

9. Booking Management Page
- Student view:
  - pending bookings
  - approved bookings
  - completed sessions

- Tutor view:
  - incoming requests
  - accept/reject buttons

-------------------------------------

10. User Profile Page
- view profile
- edit profile
- show tutor status

-------------------------------------

⚠️ DO NOT INCLUDE:
- payments
- notifications
- exam packages
- extra advanced features

Keep it focused on core system only.

-------------------------------------

🧠 UX REQUIREMENTS:
- Clear navigation between pages
- Consistent layout across screens
- Reusable components (cards, buttons, navbar)
- Logical user flow:
  Login → Dashboard → Search → Profile → Booking

-------------------------------------

🎯 OUTPUT REQUIREMENT:
- High-fidelity UI (not wireframes)
- Clean, modern, production-ready design
- Consistent design system
- Use realistic content and labels

-------------------------------------

Think like a real product designer, not just a UI generator.
